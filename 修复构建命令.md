# 修复构建命令 - JAVA_HOME 路径问题

## 🔍 问题原因

JAVA_HOME 路径包含空格，需要用引号包裹。

## ✅ 正确的构建命令

### 方法一：在命令提示符中执行（推荐）

**按顺序执行以下命令**：

```cmd
cd C:\Users\hp\WorkBuddy\Claw\BadmintonAndroid
```

然后**双击运行**：
```
C:\Users\hp\WorkBuddy\Claw\BadmintonAndroid\find_jdk.bat
```

这个脚本会自动找到您的 JDK 路径，并显示正确的命令。

---

### 方法二：手动查找并设置

**第一步：找到 JDK 路径**

1. 打开文件资源管理器
2. 进入 `C:\Program Files\Eclipse Adoptium\`
3. 查看有哪些文件夹（例如：`jdk-17.0.18-hotspot`）
4. 复制完整路径，例如：`C:\Program Files\Eclipse Adoptium\jdk-17.0.18-hotspot`

**第二步：验证 Java 是否在该路径**

打开命令提示符，执行：
```cmd
"C:\Program Files\Eclipse Adoptium\jdk-17.0.18-hotspot\bin\java.exe" -version
```

如果显示版本信息，说明路径正确。

**第三步：设置环境变量并构建**

```cmd
set "JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-17.0.18-hotspot"
set "PATH=%JAVA_HOME%\bin;%PATH%"
cd C:\Users\hp\WorkBuddy\Claw\BadmintonAndroid
gradlew.bat assembleDebug
```

**重要**：注意第一行命令中，等号两边的值都用引号包裹！

---
