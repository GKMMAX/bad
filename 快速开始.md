## 三种获取 APK 的方法

### 方法一：使用 Android Studio（最简单，推荐）

**适合人群**: 普通用户，不熟悉命令行

**步骤**:

1. **下载 Android Studio**
   - 访问: https://developer.android.com/studio
   - 下载最新版本（约 1GB）
   - 运行安装程序

2. **打开项目**
   - 启动 Android Studio
   - File -> Open
   - 选择: `c:/Users/hp/WorkBuddy/Claw/BadmintonAndroid`

3. **等待同步**
   - 首次打开会自动下载 Gradle（可能需要 10-30 分钟）
   - 等待右下角 "Syncing" 完成

4. **构建 APK**
   - 菜单: Build -> Build Bundle(s) / APK(s) -> Build APK(s)
   - 等待构建完成（5-10 分钟）

5. **获取 APK**
   - 构建完成后会弹窗，点击 "locate"
   - 或手动打开: `app/build/outputs/apk/debug/app-debug.apk`

**优点**:
- ✅ 最简单，完全图形化
- ✅ 自动下载所有需要的工具
- ✅ 包含模拟器和调试功能
- ✅ 官方工具，稳定可靠

**缺点**:
- ⚠️ 需要下载大文件（约 1GB）
- ⚠️ 首次配置需要较长时间

### 方法二：安装 JDK 17 并使用构建脚本

**适合人群**: 有一定技术基础，想要快速构建

**步骤**:

1. **下载并安装 JDK 17**
   - 下载: https://adoptium.net/temurin/releases/?version=17
   - 选择 Windows x64 版本
   - 安装并记住安装路径（如: `C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot`）

2. **配置环境变量**
   ```cmd
   # 设置 JAVA_HOME
   setx JAVA_HOME "C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot"

   # 添加到 PATH
   setx PATH "%PATH%;%JAVA_HOME%\bin"
   ```

   **注意**: 设置环境变量后需要重新打开命令行窗口

3. **验证安装**
   ```cmd
   java -version
   ```
   应该显示 Java 17 的版本信息

4. **运行构建脚本**
   ```cmd
   cd c:/Users/hp/WorkBuddy/Claw/BadmintonAndroid
   build_apk_debug.bat
   ```

5. **获取 APK**
   - 脚本会自动打开文件夹
   - 位置: `app/build/outputs/apk/debug/app-debug.apk`

**优点**:
- ✅ 快速，只需下载 JDK（约 200MB）
- ✅ 一键构建，自动化脚本
- ✅ 可重复使用

**缺点**:
- ⚠️ 需要手动配置环境变量
- ⚠️ 不包含 Android Studio 的可视化工具

### 方法三：使用 GitHub Actions 在线构建（无需本地环境）

**适合人群**: 有 GitHub 账号，熟悉 Git

**步骤**:

1. **安装 Git**（如未安装）
   - 下载: https://git-scm.com/download/win
   - 安装并使用默认选项

2. **初始化 Git 仓库**
   ```cmd
   cd c:/Users/hp/WorkBuddy/Claw
   git init
   git add BadmintonAndroid
   git commit -m "Initial commit"
   ```

3. **推送到 GitHub**
   - 在 GitHub 创建新仓库
   - 添加远程仓库:
     ```cmd
     git remote add origin https://github.com/your-username/BadmintonAndroid.git
     git push -u origin main
     ```

4. **设置 GitHub Actions**
   - 在项目中创建 `.github/workflows/build.yml`
   - 复制以下内容:

   ```yaml
   name: Build APK

   on:
     push:
       branches: [ main ]
     pull_request:
       branches: [ main ]
     workflow_dispatch:

   jobs:
     build:
       runs-on: ubuntu-latest

       steps:
       - uses: actions/checkout@v3

       - name: Set up JDK 17
         uses: actions/setup-java@v3
         with:
           java-version: '17'
           distribution: 'temurin'

       - name: Grant execute permission for gradlew
         run: chmod +x gradlew
         working-directory: ./BadmintonAndroid

       - name: Build with Gradle
         run: ./gradlew assembleDebug
         working-directory: ./BadmintonAndroid

       - name: Upload APK
         uses: actions/upload-artifact@v3
         with:
           name: app-debug
           path: BadmintonAndroid/app/build/outputs/apk/debug/*.apk
   ```

5. **触发构建**
   - 推送代码到 GitHub
   - 访问 GitHub 仓库的 Actions 页面
   - 点击 "Run workflow"

6. **下载 APK**
   - 构建完成后，在 Actions 页面找到构建记录
   - 下载 Artifacts 中的 app-debug.zip

**优点**:
- ✅ 无需本地安装任何工具
- ✅ 免费使用
- ✅ 可以设置自动构建
- ✅ 可以作为版本发布

**缺点**:
- ⚠️ 需要 GitHub 账号
- ⚠️ 首次配置稍复杂
- ⚠️ 构建时间较长（10-15 分钟）
